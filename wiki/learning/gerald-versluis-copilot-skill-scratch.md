# Creating a GitHub Copilot Skill from Scratch

**Last verified:** 2026-05-19

## Summary
Gerald Versluis demonstrates how to write, package, and publish a custom GitHub Copilot Skill from zero. Covers manifest files, prompt engineering, publishing to GitHub, and versioning best practices.

## What You'll Learn
- A Copilot Skill is a **declarative prompt package** with a manifest
- The manifest defines triggers, context rules, and output formats
- Skills can be shared privately within an org or published to the GitHub marketplace
- Versioning and testing are critical for team-wide adoption

## Workflow Covered
1. Writing the skill manifest (JSON/YAML)
2. Defining prompt templates and context rules
3. Testing locally with the Copilot CLI
4. Packaging and publishing to GitHub
5. Versioning and update strategies

## Key Takeaways
- A Copilot Skill is a declarative prompt package with a manifest.
- Versioning and testing are critical for team-wide adoption.
- Skills can be shared privately or published to the GitHub marketplace.
- The manifest is the contract between the skill and the Copilot engine.

## Recommended Audience
Teams looking to standardise Copilot behaviour across repositories and organizations.
