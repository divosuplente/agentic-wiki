## Summary
A deep-dive comparison of four specification-driven development (SDD) tools: GSD (Get Shit Done), GitHub Spec Kit, OpenSpec, and Taskmaster AI. The article maps where each tool diverges in philosophy—ranging from lightweight CLI wrappers to full orchestration frameworks—and outlines when to pick each for agentic workflows.

## Tool Comparison
| Tool | Philosophy | Best For |
|------|-----------|----------|
| **GSD** | Speed and minimal ceremony | Teams who want to ship fast with minimal overhead |
| **Spec Kit** | Tight GitHub integration | Teams already using GitHub issues, PRs, and projects |
| **OpenSpec** | Language-agnostic, no dependencies | Teams wanting universal specs without API keys or MCP |
| **Taskmaster AI** | Persistent planning and memory | Complex projects requiring long-range agent coordination |

## Key Takeaways
- GSD prioritises speed and minimal ceremony.
- Spec Kit integrates tightly with GitHub repos and issue tracking.
- OpenSpec is language-agnostic and requires no API keys.
- Taskmaster AI adds persistent planning and memory layers.
- Pick based on your team's existing toolchain and complexity needs.
