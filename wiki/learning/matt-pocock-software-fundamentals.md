## Summary
Matt Pocock makes the case that AI coding assistants highlight, rather than replace, the need for strong software fundamentals. Understanding types, architecture, and debugging is what separates effective AI-assisted engineers from passive consumers.

## Core Argument
- **Types and contracts** become *more* important when an agent generates code — you need to verify what the agent produced
- **Debugging** is the last mile that AI cannot reliably own
- **Fundamentals multiply the impact** of any tool you adopt; weak fundamentals make AI a crutch
- **The AI paradox**: the better you are at coding, the more value you extract from AI assistants

## Key Takeaways
- Types and contracts become more important when an agent generates code.
- Debugging skills are the last mile that AI cannot reliably own.
- Fundamentals multiply the impact of any tool you adopt.
- AI makes strong engineers stronger; it makes weak engineers dependent.
