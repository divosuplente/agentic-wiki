## Summary
Mo Bitar argues that the current wave of AI coding tools risks deskilling developers by removing the need to understand the code being produced. He discusses the tension between velocity and craft, and why preserving deep understanding matters.

## Core Argument
- **Velocity vs. craft** — rapid code generation can outpace understanding
- **Deskilling risk** — when the tool does the thinking, the human stops practicing
- **Debugging as the last mile** — AI can write code but struggles to debug it; humans who haven't practiced debugging can't step in
- **Tool design matters** — augmentation preserves skills; replacement erodes them

## Key Takeaways
- Rapid code generation can outpace understanding.
- Preserving debugging and architectural reasoning skills remains essential.
- Tool design should favour augmentation over replacement.
- The best AI-assisted developers are those who could write the code themselves.

## Recommended Audience
Engineers and engineering managers concerned with long-term skill retention.
