# Overview

> **TL;DR:** This wiki covers AI coding agents (OpenCode, Claude Code, Copilot), reusable skills, orchestration harnesses, and spec-driven development standards. Start with [your role](README.md#pick-your-path) or jump to [learning paths](learning/paths/README.md).


The landscape of AI-assisted development in 2026 is defined by a growing ecosystem of specialized agents, reusable skills, and orchestration tools. At the core are coding agents such as [OpenCode](./execution-surfaces/opencode.md), [Claude Code](./execution-surfaces/claude-code.md), [Pi](./execution-surfaces/pi-coding-agent.md), and [Warp](./execution-surfaces/warp.md), augmented by local intelligence layers like [CodeGraph](./protocols/codegraph.md) and [Graphify](./protocols/graphify.md), each offering distinct approaches to AI-native development: OpenCode as an open-source CLI with deep IDE integration and multi-model support; Pi as a lightweight, package-based agent with a composable skill ecosystem; and Warp as a terminal reimagined for the AI era with inline assistance and collaborative notebooks. Complementing these agents, CLI proxies such as [RTK](./ecosystem/rtk.md) and [snip](./ecosystem/snip.md) filter and compress shell output before it reaches the LLM, with snip standing out as an RTK alternative focused on YAML-based extensibility and 126 built-in declarative filters. Alongside these, a new wave of AI-native terminals and CLI proxies has emerged — [cmux](./execution-surfaces/cmux.md) brings notification rings and vertical tabs to macOS for agent multitasking, while [Terax](./execution-surfaces/terax.md) offers a cross-platform Agent Development Environment with integrated editor, file explorer, and AI side-panel. Alongside these, GitHub Copilot remains a dominant force, with a rich community of [skills](skills/awesome-copilot.md) and custom instructions extending its behavior across stacks.

The skill ecosystem has matured into a formal packaging and distribution layer. Community hubs like [The Agent Skills Directory](skills/agent-skills-directory.md) and [Vercel Skills](skills/vercel-skills.md) provide CLI-installable packages, while specialized repos such as [Community Skills](skills/plainconcepts-skills.md), [midudev/autoskills](skills/midudev-autoskills.md), and [francho-plain/francho-copilot](skills/francho-copilot.md) offer tailored instructions for specific frameworks and organizations. Beyond simple prompts, skills now address operational concerns—[Caveman](skills/caveman.md) optimizes token usage, [Opencode Dynamic Context Pruning](skills/opencode-dynamic-context-pruning.md) reduces context window pressure, and dedicated packages cover everything from [Azure DevOps PR reviews](skills/azure-devops-github-copilot-pr-review.md) to [FastAPI best practices](skills/fastapi.md). Recently, industry leaders have entered the skill space directly: Google's [Modern Web Guidance](skills/modern-web-guidance.md) provides an evergreen, expert-vetted package that guides agents across accessibility, performance, privacy, PWAs, and Baseline conformance, bridging the gap between browser vendor expertise and AI-assisted development.

Multi-agent orchestration is maturing from novelty to necessity. Harnesses such as [gastown](./governance/gastown.md), [claude-task-master](./governance/claude-task-master.md), [opencode-ensemble](./governance/opencode-ensemble.md), and [Oh My OpenAgent](./governance/oh-my-openagent.md) enable teams of specialized agents to work in parallel, delegate tasks, and message each other. [AgentCraft](./governance/agentcraft.md) even reimagines orchestration as an RTS game interface. [Paseo](./governance/paseo.md) extends this to mobile, while [spike-openspec-flow](./governance/spike-openspec-flow.md) ties orchestration directly to spec-driven development workflows. [Herdr](./governance/herdr.md) brings orchestration inside the terminal itself with tmux-style persistence and semantic agent state tracking. These tools reflect a broader shift: the unit of computation is no longer a single agent, but an ensemble.

Spec-driven development (SDD) provides the structural backbone for this shift. Frameworks like [OpenSpec](./specs/openspec.md) define lightweight, API-key-free specifications for coding agents, while [Spec Kit](./specs/spec-kit.md) and [BMAD-METHOD](./specs/bmad-method.md) offer GitHub-native and team-oriented approaches respectively. [ASDLC.io](./specs/asdlc.md) extends SDD to industrial agent-driven models with Factory Architecture, Standardized Parts, and Quality Control. [Get Shit Done](./specs/get-shit-done.md) prioritizes pragmatism, and [Squad](./specs/squad.md) focuses on agent-team formation patterns. Together these standards ensure that agentic workflows remain consistent, reviewable, and scalable across projects.

The learning curve is steep but well-documented. From [BIG School's AI Development Course](learning/big-school-ai-dev-course.md) and [Microsoft Learn Labs](learning/microsoft-learn-labs.md) to the [Kaggle 5-Day AI Agents curriculum](learning/kaggle-5-day-ai-agents.md), developers can build competency in tool use, planning, and orchestration. [The Agentic SDLC Handbook](learning/agentic-sdlc-handbook.md) provides a comprehensive PROSE framework for building software with AI agents. Voices like [Karpathy](learning/karpathy-vibe-coding-agentic.md), [Steve Yegge](learning/steve-yegge-manifesto-latent-space.md), and [Mo Bitar](learning/mo-bitar-no-skill-ai-coding.md) warn against [comprehension debt](learning/comprehension-debt.md) and deskilling, arguing that strong fundamentals multiply the impact of any tool.

The community sits at the intersection of this ecosystem, maintaining [skill repositories](skills/plainconcepts-skills.md) and contributing through [community events](./ecosystem/one-to-all-plain-agents-jam.md). Multiple agents fit into daily workflows—OpenCode for rapid prototyping and refactoring, Warp for pair-programming and terminal-native tasks, and Pi for focused, skill-heavy code generation. By combining SDD frameworks like OpenSpec with orchestration harnesses and a curated skill library, the goal is to scale AI-assisted development without losing engineering craftsmanship.
