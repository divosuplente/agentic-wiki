## Description
The GSD framework emphasizes pragmatic, results-oriented agent workflows to ship software efficiently. Prioritizes speed and minimal ceremony in spec-driven development.

## Philosophy
- **Speed first** — minimal boilerplate, maximum shipping
- **Pragmatic specs** — enough structure to guide agents, not enough to slow them down
- **Results-oriented** — measure by working code, not documentation completeness
- **Agent-native** — designed for how AI agents work, not how humans write specs

## Positioning
Among SDD tools, GSD sits at the lightweight end:
- Less ceremony than Spec Kit
- Faster to start than OpenSpec
- Fewer guardrails than Taskmaster AI
